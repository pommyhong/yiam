# Yiam — Edge Cases

จัดกลุ่มตามจุดที่พังได้จริง · ✅ = ออกแบบ state ไว้แล้ว / ⚠️ = ต้องตัดสินใจ

---

## A. Input — ตอนพิมพ์ amount
```
ใส่ 0 หรือเว้นว่าง            → ปุ่มบันทึก disabled (เทา) ✅
ใส่ทศนิยมเกิน 2 ตำแหน่ง        → ตัดเหลือ 2 / บล็อกตั้งแต่พิมพ์ ⚠️
กด . ซ้ำ / . นำหน้า           → keypad บล็อกจุดที่ 2, ".5" → "0.5" ✅
จำนวนมหาศาล (เช่น 9 หลัก)     → จัดกลุ่มหลัก (1,000,000) + ตัวเลขย่อขนาดไม่ให้ล้น ⚠️
ติดลบ / paste ตัวอักษร        → keypad เฉพาะตัวเลข ไม่มีปัญหา ✅
note ยาวมาก                  → จำกัดความยาว + truncate ในรายการ (…) ✅
ชื่อหมวด/merchant ยาว         → 1 บรรทัด + ellipsis, amount ไม่ขยับ ✅
```

## B. Category
```
ไม่เลือกหมวด                 → ตัดสินใจ: auto "อื่นๆ" (ลื่น) vs บังคับเลือก ⚠️
หมวดที่ลบไปแล้วแต่มีรายการเก่าผูก → รายการเก่าต้องไม่หาย → fallback "อื่นๆ"/เก็บชื่อเดิมไว้ ⚠️
หมวดเยอะมากในตัวเลือก          → search / เรียงตามใช้บ่อย ⚠️
```

## C. Date / เวลา
```
เลือกวันอนาคต                 → อนุญาตไหม? (จดล่วงหน้า) — แนะนำเตือนแต่ให้ผ่าน ⚠️
วันข้ามเดือน (28→1)          → รายการต้องเข้าเดือนถูกในสรุป + analytics ✅
timezone / เที่ยงคืน          → ยึด local time ของเครื่อง, "วันนี้" = local ✅
รายการย้อนหลังไกลมาก          → จัดกลุ่มตามเดือน, ไม่พังการเลื่อน ✅
```

## D. Analytics — จุดพังเยอะสุด
```
ยังไม่มีข้อมูลเลย             → empty: "ยังไม่มีข้อมูลพอแสดงกราฟ" ✅
ช่วงเวลานั้นไม่มีรายการ        → กราฟว่าง + ข้อความ ไม่ใช่หน้าโล่งงงๆ ✅
มีแค่ 1 รายการ               → กราฟ/% ยังต้องอ่านได้ (ไม่ใช่แท่งเดียวเต็มจอ) ⚠️
รวม = 0 ตอนคำนวณ %           → กัน division by zero → แสดง "—" ไม่ใช่ NaN/∞ ✅
มีแต่รายรับ ไม่มีรายจ่าย       → analytics จ่ายต้องไม่พัง (แสดง 0 / สลับดูรับ) ⚠️
หมวดเยอะมาก (15+)            → chart รวม "อื่นๆ" + top N, ไม่ยัดทุกหมวด ⚠️
หมวดเดียวกิน 100%            → donut เต็มวง + label ยังอ่านออก ✅
ตัวเลขติดลบสุทธิ (จ่าย>รับ)    → แสดงเครื่องหมาย/สีให้ชัด ✅
```

## E. Lifecycle — แก้/ลบ/ข้อมูล
```
ลบรายการ                    → confirm ก่อนลบ + undo (toast) ✅
ลบรายการสุดท้าย              → กลับสู่ empty state สวยๆ ✅
แก้ amount/หมวด              → สรุป + analytics อัปเดตทันที ✅
ปิดแอปกลางการกรอก            → draft หาย หรือเก็บ? — แนะนำทิ้ง (รายการสั้น) ⚠️
ข้อมูลเยอะมาก (1000+ รายการ)  → list ต้องลื่น (virtualize) + สรุปไม่ช้า ⚠️
ลบแอป/เปลี่ยนเครื่อง          → ข้อมูล local หาย → สื่อสารว่าไม่มี cloud (v1) ⚠️
รายการซ้ำ (กดบันทึก 2 ที)     → กัน double-submit (disable ปุ่มตอน save) ✅
```

## F. Display / i18n
```
สกุลเงิน ฿ + จัดหลัก          → ฿1,234.50 รูปแบบไทยสม่ำเสมอทุกที่ ✅
ตัวเลขยาวในการ์ดแคบ           → ย่อขนาด font / truncate ขวา ✅
ภาษาไทยตัวสูง-ต่ำ (ปํ ญ ฎ)    → line-height เผื่อ ไม่ตัดหัว/หาง ✅
dark mode                   → v1 ไม่ทำ (mono อยู่แล้ว) แต่ token เผื่อไว้ ⚠️
```

## G. Accessibility
```
กราฟสื่อด้วยสีอย่างเดียว       → mono อยู่แล้ว → ต้องมี label/ตัวเลขกำกับเสมอ ✅
contrast ดำบนขาว             → ผ่าน WCAG สบาย ✅
touch target                → ปุ่ม/row ≥ 44pt ✅
disabled button             → อย่าใช้สีจางอย่างเดียว → มี state ชัด ⚠️
keypad ตัวเลขเล็ก            → ปุ่มใหญ่พอกดด้วยนิ้วโป้ง ✅
```

## Resolved decisions (เคาะแล้ว 2026-06-17)
เกณฑ์: ยึด metric "บันทึก <5 วิ · ≤4 tap" + minimal — อะไรเพิ่ม friction ตัดทิ้ง
```
1. ไม่เลือกหมวด     → auto "อื่นๆ" (เลือกแก้ทีหลังได้) — ไม่บังคับ
2. วันอนาคต        → อนุญาต, default = วันนี้ (จดล่วงหน้าได้)
3. หมวดถูกลบ       → N/A v1 (หมวด preset แก้ไม่ได้) → ไม่มี orphan; v2 ค่อย fallback "อื่นๆ"
4. draft ปิดกลางคัน → ทิ้ง (discard) — รายการสั้น ไม่เก็บ draft
5. ไม่มี cloud v1   → บรรทัดเงียบใน empty/onboarding ("ข้อมูลเก็บในเครื่องนี้") ไม่เด่น
6. analytics น้อย/เอียง → degrade:
     • < 3 รายการ → ซ่อนกราฟ แสดง list + ยอดรวมแทน + "ข้อมูลยังน้อย"
     • รวม = 0    → "—" ไม่ใช่ NaN
     • > 6 หมวด   → top 6 + รวมที่เหลือเป็น "อื่นๆ"
     • 1 หมวด 100% → donut เต็มวง + label ชัด
```
